<objective>
为 Go 后端补一层轻量日志门面，把 info/warn 与 error/fatal 明确分流到 stdout / stderr，并先覆盖最容易混杂的服务入口、调度器与标签处理热点。

Purpose: 让本地运行、systemd/docker 日志采集、错误排查都能快速分辨“正常进度”与“异常失败”，避免当前 `log.SetOutput(os.Stdout)` + 各处 `[WARN]/[ERROR]` 字符串混排。

Output: 一个可复用的 `internal/platform/logging` 辅助包，以及对服务启动、运行时调度器、summary/tagging 热点文件的最小替换。
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| backend process → stdout collector | 正常运行日志会被终端、Docker、systemd 等采集 |
| backend process → stderr collector | 异常/崩溃日志会被告警或错误采集链路单独处理 |
| domain data → log line | 标签名、订阅源名等动态内容进入日志文本 |
