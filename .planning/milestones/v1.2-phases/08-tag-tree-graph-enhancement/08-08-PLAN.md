The root cause of "empty black dialog" is a combination of issues:

**Problem 1 — Teleport creates a second overlay:** TagMergePreview uses `<Teleport to="body">` with its own dark overlay (`.tag-merge-overlay` with `background: rgba(8, 12, 18, 0.7)` and `backdrop-filter: blur(10px)`). When rendered inside GlobalSettingsDialog (which already has its own overlay `bg-black/50`), the user sees two stacked overlays — the inner one is the "black dialog" with no content (because state is idle, as fixed in Plan 07 Task 2).

**Problem 2 — Idle state shows nothing:** Even with the immediate watch fix from Plan 07, during the brief moment before the scan starts, the component shows the `.tag-merge-modal` div with no content (idle state has no template branch).

**Fix — Add a `standalone` prop to TagMergePreview:**

1. In `TagMergePreview.vue`, add a `standalone` prop (default `true` for backward compat):

   ```typescript
   interface Props {
     visible: boolean
     scopeCategoryId?: string | null
     scopeFeedId?: string | null
     standalone?: boolean  // when false, skip Teleport and overlay
   }

   const props = withDefaults(defineProps<Props>(), {
     scopeCategoryId: null,
     scopeFeedId: null,
     standalone: true,
   })
   ```

2. Change the template to conditionally use Teleport:

   Replace the current `<Teleport to="body">` wrapper with:
   ```html
   <component :is="props.standalone ? 'Teleport' : 'div'" v-bind="props.standalone ? { to: 'body' } : {}">
     <div v-if="visible" class="tag-merge-overlay" :class="{ 'tag-merge-overlay--inline': !props.standalone }" @click.self="handleClose">
       <div class="tag-merge-modal" :class="{ 'tag-merge-modal--inline': !props.standalone }">
         <!-- existing content unchanged -->
       </div>
     </div>
   </component>
   ```

   NOTE: Using a dynamic component approach may not work cleanly with Teleport. Instead, use `v-if`/`v-else`:
   ```html
   <!-- Standalone mode: teleport to body with overlay -->
   <Teleport v-if="props.standalone" to="body">
     <div v-if="visible" class="tag-merge-overlay" @click.self="handleClose">
       <div class="tag-merge-modal">
         <!-- ... existing modal content ... -->
       </div>
     </div>
   </Teleport>
   <!-- Inline mode: render directly without teleport or overlay -->
   <div v-else>
     <div v-if="visible" class="tag-merge-inline">
       <div class="tag-merge-inline__content">
         <!-- ... same content as modal but with inline styles ... -->
       </div>
     </div>
   </div>
   ```

   This is too much duplication. Better approach: Extract the inner content and conditionally wrap it.

   **Simplest fix:** Just use `v-if` on Teleport wrapper and render the same content without overlay when not standalone:

   ```html
   <template v-if="props.standalone">
     <Teleport to="body">
       <div v-if="visible" class="tag-merge-overlay" @click.self="handleClose">
         <div class="tag-merge-modal">
           <TagMergeContent :state="state" :candidates="candidates" ... />
         </div>
       </div>
     </Teleport>
   </template>
   <template v-else>
     <div v-if="visible" class="tag-merge-modal tag-merge-modal--inline">
       <TagMergeContent :state="state" :candidates="candidates" ... />
     </div>
   </template>
   ```

   This still requires extracting content into a sub-component, which is complex.

   **Simplest pragmatic fix:** Remove the Teleport entirely and use CSS to control overlay behavior. Add a prop `inline` that when true:
   - Skips the overlay background
   - Skips the click.self dismiss
   - Renders the modal content directly

   ```html
   <Teleport v-if="!props.inline" to="body">
     <div v-if="visible" class="tag-merge-overlay" @click.self="handleClose">
       <div class="tag-merge-modal">
         <!-- all existing content here -->
       </div>
     </div>
   </Teleport>
   <div v-else-if="visible" class="tag-merge-modal tag-merge-modal--inline">
     <!-- all existing content here, duplicated -->
   </div>
   ```

   **ACTUALLY SIMPLEST FIX (chosen):** Don't duplicate. Just conditionally wrap:

   ```html
   <Teleport to="body" :disabled="!props.standalone">
     <div v-if="visible" :class="props.standalone ? 'tag-merge-overlay' : 'tag-merge-inline'" @click.self="props.standalone ? handleClose() : undefined">
       <div :class="props.standalone ? 'tag-merge-modal' : 'tag-merge-inline__content'">
         <!-- ALL existing content stays the same -->
       </div>
     </div>
   </Teleport>
   ```

   Teleport with `:disabled="true"` renders in-place without teleporting. This is the cleanest approach.

3. Add the `standalone` prop with default `true`.

4. Add inline styles:
   ```css
   .tag-merge-inline {

   .tag-merge-inline__content {

     :standalone="false"
     @merged="handleMerged"
     @close="activeTab = 'backend-queues'"
   />
   ```
