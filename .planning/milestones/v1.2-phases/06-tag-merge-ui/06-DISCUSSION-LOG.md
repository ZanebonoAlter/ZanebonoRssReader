## 预览界面布局

| Option | Description | Selected |
|--------|-------------|----------|
| 卡片列表 | 每对一个卡片，显示源/目标名称、相似度、文章数，可点击展开查看关联文章标题。适合少量候选对 | ✓ |
| 数据表格 | 紧凑的表格行，列显示源名称、目标名称、相似度、源文章数、目标文章数、操作按钮。适合大量候选对 | |
| 双栏布局 | 左右两栏，左侧显示候选列表（点击选中），右侧显示选中对的详情和操作 | |

**User's choice:** 卡片列表
**Notes:** 用户认为卡片列表适合预期少量候选对（≤50 对）的场景，可展开查看详情

---

## 名称编辑方式

| Option | Description | Selected |
|--------|-------------|----------|
| 内联编辑 | 在卡片上直接显示编辑按钮，点击后输入框出现，可修改名称后保存或取消。操作就在预览界面完成 | ✓ |
| 弹窗编辑 | 点击编辑按钮后弹出对话框，输入新名称后确认。与预览界面分离，更清晰 | |
| 合并后改名 | 合并前用户不能修改名称，合并后可进入标签详情页改名。流程简单但不够灵活 | |

**User's choice:** 内联编辑
**Notes:** 操作就在预览界面完成，无需弹窗，用户体验流畅

---

## 确认流程

| Option | Description | Selected |
|--------|-------------|----------|
| 逐对确认 | 卡片上显示「合并」和「跳过」按钮，逐对操作。没有批量合并按钮，确保用户审视每个对 | |
| 逐对 + 批量合并 | 除逐对操作外，顶部增加「全部合并」按钮，一键处理所有候选对。适合用户信任自动推荐 | ✓ |
| 支持撤销 | 合并后提供撤销按钮，可恢复被合并的标签。但技术上复杂（需恢复引用关系） | |

**User's choice:** 逐对 + 批量合并
**Notes:** 用户指出这是手动干预功能，自动合并已由 scheduler 处理。UI 应支持逐对审视 + 批量合并选项

## 结果展示

| Option | Description | Selected |
|--------|-------------|----------|
| 合并汇总 | 合并完成后显示汇总：成功 N 个、跳过 M 个、失败 K 个，并列出被合并标签名称 | ✓ |
| 跳转标签详情 | 合并后自动跳转到目标标签详情页，可查看合并后的文章列表 | |
| Toast 通知 | 显示成功 Toast 通知，用户继续在预览界面处理剩余候选对 | |

**User's choice:** 合并汇总
**Notes:** 显示完整汇总信息，包括被合并的标签名称列表
