3. **Update GenerateEmbedding** in `embedding.go`:
   - After generating embedding, store in both `Vector` (legacy JSON, backward compat) AND `EmbeddingVec` (pgvector format — `[0.1,0.2,...]` string representation)
   - Pgvector accepts `[0.1,0.2,0.3]` format for vector column inserts

4. **Replace FindSimilarTags** in `embedding.go`:
   - Replace the current Go-side loop (load all embeddings → json.Unmarshal each → CosineSimilarity) with a single SQL query using pgvector `<=>` operator:
   ```sql
   SELECT topic_tag_embeddings.*, topic_tags.*
   FROM topic_tag_embeddings
   JOIN topic_tags ON topic_tags.id = topic_tag_embeddings.topic_tag_id
   WHERE topic_tags.category = ?
     AND topic_tag_embeddings.embedding IS NOT NULL
     AND topic_tags.status = 'active'
   ORDER BY topic_tag_embeddings.embedding <=> $1::vector
   LIMIT ?
   ```
   - The `$1::vector` parameter should be the query embedding in pgvector string format `[0.1,0.2,...]`
   - Use `database.DB.Raw()` with GORM `Scan()` for the results
   - Cosine distance from `<=>` gives 0 for identical, 2 for opposite; similarity = 1 - distance
   - Map the SQL results back to `[]TagCandidate` with computed similarity scores
   - Remove the Go-side CosineSimilarity call loop entirely from this function
   - Keep `airouter.CosineSimilarity` function intact — it may be used elsewhere

5. **Update getEmbeddingModel** in `embedding.go` (per D-11, INFRA-02):
   - Read `provider.Model` field. If the provider configured via airouter has a model set, use it
   - The `AIProvider.Model` field stores the model name (e.g., "text-embedding-3-small")
   - If `provider.Model` is empty, fall back to "text-embedding-ada-002" as hardcoded default
   - This replaces the current hardcoded return "text-embedding-ada-002"
