<!-- Component state machine -->
State flow: idle → scanning → preview → summary → close → idle
- idle: nothing visible
- scanning: modal open, loading spinner
- preview: cards displayed, user can edit/merge/skip
- summary: merge results displayed
- close: emit events, reset to idle

Create a single complete Vue component implementing all D-01 through D-04:

**Script setup:**

1. Imports:
   ```typescript
   import { ref, computed, watch } from 'vue'
   import { Icon } from '@iconify/vue'
   import { useTagMergePreviewApi } from '~/api/tagMergePreview'
   import type { TagMergeCandidate, MergeSummary } from '~/types/tagMerge'
   ```

2. Props and emits:
   ```typescript
   interface Props { visible: boolean }
   const props = defineProps<Props>()
   const emit = defineEmits<{ close: []; merged: [summary: MergeSummary] }>()
   ```

3. State refs:
   - `state`: `'idle' | 'scanning' | 'preview' | 'summary'`
   - `candidates`: `TagMergeCandidate[]`
   - `expandedIds`: `number[]` — which cards have articles expanded
   - `editingId`: `number | null` — which card is in edit mode
   - `editedName`: `string` — the name being edited
   - `customNames`: `Map<number, string>` — saved custom names per candidate
   - `mergedIds`: `number[]` — successfully merged candidate IDs
   - `skippedIds`: `number[]` — skipped candidate IDs
   - `failedIds`: `number[]` — failed merge IDs
   - `mergingIds`: `number[]` — currently merging (loading state)
   - `batchMerging`: `boolean`
   - `mergeSummary`: `MergeSummary | null`
   - `error`: `string | null`

4. Computed `visibleCandidates`: filter out merged + skipped from candidates

5. Core functions:
   - `startScan()`: set state=scanning, call `scanMergePreview({ limit: 50, includeArticles: true })`, on success set candidates + state=preview, on error show message
   - `watch(() => props.visible)`: when visible becomes true and state=idle, call startScan
   - `toggleExpand(id)`: toggle id in expandedIds array
   - `startEdit(candidate)`: set editingId + editedName (default = customNames.get or targetLabel)
   - `saveEdit(candidate)`: save to customNames map, clear editingId
   - `cancelEdit()`: clear editingId
   - `skipCandidate(id)`: push to skippedIds
   - `mergeSingle(candidate)`: push to mergingIds, call `mergeTagsWithCustomName({ sourceTagId, targetTagId, newName: customNames.get or targetLabel })`, on success push to mergedIds, on fail push to failedIds, remove from mergingIds
   - `batchMerge()`: set batchMerging=true, loop visibleCandidates and call mergeSingle for each, then buildSummary + state=summary
   - `buildSummary()`: compute MergeSummary from mergedIds/skippedIds/failedIds + candidates
   - `handleClose()`: emit 'merged' with summary if exists, emit 'close', reset all state

**Template (single Teleport modal):**

6. `<Teleport to="body">` with fixed inset-0 overlay:
   - **Scanning state**: centered loading spinner + "正在扫描相似标签..." text
   - **Preview state**:
     - Header: title "标签合并预览 (N 对候选)" + batch merge button "全部合并" + close button
     - Empty state (v-if no visibleCandidates): "没有发现相似标签"
     - Card list (v-for visibleCandidates):
       - Card header: sourceLabel → (arrow icon) → targetLabel (or customName if set) + edit pencil button
       - Card stats: similarity percentage badge + "N → M 文章" text
       - Inline edit (v-if editingId matches): input + save/check + cancel/close buttons
       - Expand button: toggle article titles section
       - Article titles (v-if expanded): two columns — source articles list + target articles list
       - Card actions: "合并" button (with loading when merging) + "跳过" button
   - **Summary state**:
     - Success icon + "合并完成" title
     - Stats row: merged count (green) + skipped count (yellow) + failed count (red if any)
     - Merged details list: each entry shows "sourceLabel → newLabel"
     - "完成" close button

7. Scoped styles:

   - Use the project's editorial/magazine feel, avoid generic SaaS styling
