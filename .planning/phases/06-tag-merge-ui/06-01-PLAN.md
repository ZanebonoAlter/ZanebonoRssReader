**File 1: `tag_merge_preview.go`** — Extract scan logic from auto_tag_merge.go

1. Define `TagMergeCandidate` struct:
   - SourceTagID (uint), SourceLabel (string), SourceSlug (string)
   - TargetTagID (uint), TargetLabel (string), TargetSlug (string)
   - Category (string), Similarity (float64)
   - SourceArticles (int), TargetArticles (int)

2. Define `CandidateArticle` struct:
   - ArticleID (uint), Title (string), Link (string)

3. Function `ScanSimilarTagPairs(limit int) ([]TagMergeCandidate, error)`:
   - Compute distanceThreshold = 1.0 - DefaultThresholds.HighSimilarity
   - Use the SAME pgvector cross-join SQL from auto_tag_merge.go lines 243-259
   - For each raw pair, load both tags, verify active status
   - Count articles via `article_topic_tags` table for each tag
   - Determine source/target: tag with MORE articles is target (keep), FEWER is source (merge into target). Equal → smaller ID = source (deterministic). Same logic as auto_tag_merge.go lines 311-342
   - Compute similarity = 1.0 - distance
   - Return candidates sorted by similarity DESC

4. Function `GetCandidateArticleTitles(tagID uint, limit int) ([]CandidateArticle, error)`:
   - Query: SELECT a.id, a.title, a.link FROM articles a JOIN article_topic_tags at ON a.id = at.article_id WHERE at.topic_tag_id = ? ORDER BY a.published_at DESC LIMIT ?
   - Return article ID, title, link for preview display

**File 2: `tag_merge_preview_handler.go`** — Two HTTP handlers

5. Handler `ScanMergePreviewHandler(c *gin.Context)`:
   - Parse `limit` query param (default 50, max 100)
   - Parse `include_articles` query param (default false)
   - Call `ScanSimilarTagPairs(limit)`
   - If `include_articles=true`, for each candidate call `GetCandidateArticleTitles` for both source and target tags, attach to response
   - Return `{ "success": true, "data": { "candidates": [...], "total": N } }`
   - Each candidate in JSON uses snake_case tags matching Go struct

6. Handler `MergeTagsWithCustomNameHandler(c *gin.Context)`:
   - Parse body: `{ "source_tag_id": uint, "target_tag_id": uint, "new_name": string }`
   - Validate: both IDs required, non-zero, not equal; new_name non-empty
   - Load both tags, verify exist and status != "merged"
   - If new_name differs from target.Label:
     - Update target.Label = new_name in database
     - Update target.Slug = slugify(new_name) using `topictypes.Slugify`
   - Call `MergeTags(source_tag_id, target_tag_id)` (existing function handles everything)
   - Return `{ "success": true, "data": { "source_id": N, "target_id": N, "new_label": "...", "merged_at": "..." } }`
   - Error handling: tag not found → 404, already merged → 400, DB error → 500

7. Function `RegisterTagMergePreviewRoutes(rg *gin.RouterGroup)`:
   ```go
   func RegisterTagMergePreviewRoutes(rg *gin.RouterGroup) {
       tags := rg.Group("/topic-tags")
       {
           tags.GET("/merge-preview", ScanMergePreviewHandler)
           tags.POST("/merge-with-name", MergeTagsWithCustomNameHandler)
       }
   }
   ```

Per D-05: preview only, no auto-merge. Per D-01: response includes all card fields. Per D-02: custom name supported.
