<objective>
Integrate EmbeddingService.TagMatch into findOrCreateTag so every new article tag goes through three-level semantic matching (exact → alias → embedding similarity) before creating a new tag.

Purpose: This is the core convergence mechanism. When new articles arrive, their tags are matched against existing tags via embeddings — high similarity auto-reuses existing tags, preventing tag space fragmentation (per CONV-01). Middle band skips AI judgment and creates new tags (per CONV-03, D-13).

Output: findOrCreateTag uses TagMatch for every tag resolution. Middle band degrades to creating new tags.
</objective>

Per CONV-01 (integrate TagMatch into findOrCreateTag) and CONV-03 (middle band skips AI, creates new tag):

1. **Modify findOrCreateTag** in `tagger.go`:
   - Create a package-level `var embeddingService *topicanalysis.EmbeddingService` initialized lazily
   - Add a `sync.Once` to initialize the EmbeddingService on first use (avoid import cycle issues — topicanalysis already imports models, and topicextraction importing topicanalysis is fine)
   - Replace the current logic:

   **Current flow:**
   ```
   slug+category exact match → found? update & return : create new
   ```

   **New flow (per CONV-01):**
   ```
   1. Call embeddingService.TagMatch(ctx, tag.Label, category, aliasesString)
   2. If TagMatch returns error (embedding unavailable):
      → Fall back to current slug+category exact match (preserve existing behavior)
   3. If MatchType == "exact":
      → Update existing tag (label, aliases, icon) and return it
   4. If MatchType == "high_similarity":
      → Return the existing tag WITHOUT modifying it (auto-reuse per CONV-01)
      → Optionally update aliases on the existing tag if the new tag has new aliases
   5. If MatchType == "low_similarity" || MatchType == "no_match":
      → Create new tag (current creation logic)
   6. If MatchType == "ai_judgment" (middle band, per CONV-03):
      → Skip AI judgment entirely, just create a new tag
      → This is the key change from CONV-03: "中间地带跳过 AI 判定，降级为创建新标签"
   ```

2. **Update TagMatch middle-band handling** in `embedding.go`:
   - Per CONV-03, the ai_judgment case should now set `ShouldCreate: true` instead of `false`
   - Update the comment to explain this is intentional — middle band degrades to creating new tags
   - The Candidates field in TagMatchResult is still populated (useful for logging/debugging) but no AI judgment is called

3. **Add context parameter** to findOrCreateTag:
   - Change signature from `findOrCreateTag(tag topictypes.TopicTag, source string)` to `findOrCreateTag(ctx context.Context, tag topictypes.TopicTag, source string)`
   - Update both callers: TagSummary (tagger.go:62) and tagArticle (article_tagger.go:78) to pass `context.Background()`
   - Both callers already have `context.Background()` available in their scope

4. **Add graceful degradation**:
   - If `NewEmbeddingService()` returns nil or TagMatch errors, fall back to slug+category exact match
   - Log the error with `fmt.Printf("[WARN] TagMatch failed, falling back to exact match: %v\n", err)`
   - This ensures tag creation never fails even if embedding provider is down (per D-13)
