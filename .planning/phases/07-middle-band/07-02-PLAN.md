    **1. 创建 `front/app/types/topicTag.ts`**:
    ```typescript
    export interface TagHierarchyNode {
      id: number
      label: string
      slug: string
      category: string
      icon: string
      feedCount: number
      similarityScore?: number
      children: TagHierarchyNode[]
    }

    export interface TagHierarchyResponse {
      nodes: TagHierarchyNode[]
      total: number
    }

    export interface UpdateAbstractNameRequest {
      newName: string
    }

    export interface DetachChildRequest {
      childId: number
    }
    ```

    **2. 创建 `front/app/api/abstractTags.ts`**:
    - `useAbstractTagApi()` composable 返回：
      - `fetchHierarchy(category?: string)` → GET /api/topic-tags/hierarchy?category=
      - `updateAbstractName(tagId: number, newName: string)` → PUT /api/topic-tags/:id/abstract-name
      - `detachChild(parentId: number, childId: number)` → POST /api/topic-tags/:id/detach
    - snake_case → camelCase 映射在 API 层完成
    - 复用 `apiClient` from `./client`

    **3. 创建 `front/app/features/topic-graph/components/TagHierarchy.vue`**:
    - 使用 `<script setup lang="ts">` Composition API
    - **布局**: 按类别（事件/人物/关键词）分组展示层级树
    - **递归渲染**: 每个 TagHierarchyNode 渲染为可展开/折叠的行
      - 展开按钮（如果有 children）+ 类别 icon + 标签 label + 文章数 badge
      - 子节点缩进渲染（递归组件或 v-for 递归模板）
    - **交互** (per D-04):
      - 双击抽象标签名称进入编辑模式（inline edit）
      - 编辑确认后调用 updateAbstractName API
      - 子标签旁有"分离"按钮，确认后调用 detachChild API
    - **空状态**: 无层级关系时显示提示文案
    - **样式**: 遵循项目 editorial 风格，用 Tailwind CSS v4
      - 深色背景配层级缩进线（类似文件树）
      - 展开动画用 CSS transition
      - 不使用紫色/靛蓝色 SaaS 色板
    - **加载状态**: 初始加载时显示 skeleton
