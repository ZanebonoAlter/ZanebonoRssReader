# Topics界面设计调整方案

## 核心调整点

### 1. 数据关联关系重新定义
**原设计**: AI总结包含事件/人物/关键词的概括  
**新设计**: 事件/人物/关键词直接与具体的articles关联，AI总结只是引用这些标签

### 2. 拓扑图连线交互重新设计
**原设计**: 默认展示所有连线，选中后高亮  
**新设计**: 
- 默认只展示节点，**不展示任何连线**
- 选中某个题材（标签）后，**才动态绘制该题材相关的连线**
- 连线样式需要优化，比之前更好看（考虑曲线、渐变色、动画效果）

### 3. 拓扑图下方新增时间线事件区
**位置**: 拓扑图正下方  
**内容**: 
- 按时间线展示选中题材相关的**日报内容**（即原始articles）
- 每条展示：标题、摘要、发布时间、来源feed
- 时间线形式：垂直时间轴，左侧时间，右侧内容卡片
- 支持点击展开查看全文
- 只展示与当前选中题材标签关联的日报

### 4. 右侧详情栏大幅简化
**原设计**: 展示各种相关内容，有重复  
**新设计**:
- **只展示**带有对应关键tag的articles（严格去重，不重复展示）
- **相关主题改为关键词形式**展示（纯标签云形式）
- **点击关键词仅高亮拓扑图中关联的节点**，不在右侧展开列表
- 整体保持极简，避免信息过载

### 5. AI分析功能重新定位
**展示策略**:
- **默认只展示日报内容**（原始数据，不做AI分析展示）
- 提供AI分析按钮/切换开关
- 当用户点击"AI分析"后，针对当前选中的关键元素（事件/人物/关键词）进行深度AI分析
- AI分析结果显示在独立区域（可以是时间线下方的新面板，或弹窗形式）
- 支持增量式存储，避免重复调用AI

## 需要调整的文件清单

### 前端组件调整

1. **TopicGraphCanvas.client.vue** - 拓扑图连线逻辑重构
   - 默认不渲染连线
   - 选中后动态绘制连线（带动画）
   - 优化连线样式（曲线、渐变、动画）

2. **TopicTimeline.vue** (新建) - 时间线事件区
   - 垂直时间轴布局
   - 日报内容卡片展示
   - 时间排序和筛选
   - AI分析按钮集成

3. **TopicGraphFooterPanels.vue** - 移除或简化
   - 原有历史温度面板移除
   - 整合时间线组件

4. **TopicGraphSidebar.vue** - 右侧栏简化
   - 只展示带key tag的articles
   - 去重逻辑
   - 关键词云形式展示相关主题
   - 点击只高亮拓扑节点，不展开列表

5. **TopicGraphPage.vue** - 布局和数据流调整
   - 新增时间线区域
   - 调整下方布局
   - 新的数据流管理

### 后端API调整

1. **API响应结构调整**
   - 返回与标签关联的原始articles（日报内容）
   - 不再只返回AI总结，而是返回原始articles列表
   - 按时间排序

2. **新的API端点（可选）**
   - 获取标签关联的日报列表
   - 支持时间筛选和分页

## AI分析功能的新定位

### 展示策略
1. **默认状态**: 只展示原始日报内容（不展示AI分析）
2. **用户触发**: 提供"AI分析"按钮或切换开关
3. **AI分析内容**: 针对当前选中的关键元素（事件/人物/关键词）进行深度分析
4. **展示位置**: 
   - 方案A: 时间线下方新增独立面板
   - 方案B: 弹窗/抽屉形式展示
   - 方案C: 可展开的卡片形式
5. **增量存储**: 支持分析结果缓存，避免重复调用AI
