# Topics界面设计调整方案

## 核心调整点概览

### 1. 数据展示逻辑重构
**当前问题**: 事件/人物/关键词与AI总结关联，用户看到的是概括性内容  
**调整后**: 
- 事件/人物/关键词**直接关联具体的articles**（原始日报内容）
- AI总结仅作为引用这些标签的概括
- 用户优先看到的是**真实的日报内容时间线**

### 2. 拓扑图连线交互优化
**当前问题**: 默认展示所有连线，视觉上过于复杂，选中后高亮效果不够突出  
**调整后**:
- **默认状态**: 只展示节点，**完全不显示任何连线**
- **选中题材后**: **动态绘制**与该题材相关的连线
- **视觉效果优化**:
  - 使用**贝塞尔曲线**替代直线，更柔和
  - **渐变色**（从橙色到蓝色过渡）
  - **动画效果**（连线从节点生长出来的动画）
  - **发光效果**（选中连线的光晕）

### 3. 拓扑图下方新增"日报时间线"区域
**位置**: 拓扑图正下方，作为主要内容区域  
**展示内容**:
- **垂直时间轴**形式（左侧时间，右侧内容）
- **日报内容卡片**，每条包含：
  - 标题（可点击展开全文）
  - 摘要（前100字）
  - 发布时间
  - 来源feed名称
  - 关联的标签（事件/人物/关键词标签）
- **按时间倒序**排列（最新的在最上面）
- **筛选功能**: 可以按事件/人物/关键词筛选显示

**AI分析入口**:
- 在时间线区域顶部提供"AI深度分析"按钮
- 默认收起，点击后展开AI分析面板
- AI分析内容包括：
  - 事件脉络总结
  - 关键人物关系图
  - 关键词趋势分析

### 4. 右侧详情栏精简优化
**当前问题**: 内容过多，有重复展示  
**调整后**:
- **只展示带有对应关键tag的articles**（严格去重）
  - 使用Set进行去重
  - 不再展示已经出现在时间线区域的articles
- **相关主题改为"关键词云"形式**:
  - 不再以列表形式展示
  - 使用标签云/词云形式
  - 字号大小表示关联度
- **点击关键词仅高亮拓扑图节点**:
  - 不在右侧展开任何列表
  - 只在拓扑图中高亮对应的节点
  - 提供一个"查看该关键词日报"按钮，点击后跳转到时间线并筛选

### 5. AI分析功能的重新定位
**展示策略**:
- **默认只展示日报内容**（原始articles，不展示AI分析）
- **AI分析作为增强功能**，默认收起
- **用户主动触发**: 提供"AI深度分析"按钮
- **分析对象**: 针对当前选中的关键元素（事件/人物/关键词）
- **展示位置**: 
  - 方案A: 时间线区域下方独立面板（可展开/收起）
  - 方案B: 右侧栏新增"AI分析"Tab
  - **推荐方案A**，与时间线形成对比（原始数据vsAI分析）

## 实施优先级

### P0 - 核心结构调整
1. 拓扑图连线交互改为"选中后展示"
2. 拓扑图下方新增"日报时间线"区域
3. 数据流调整：事件/人物/关键词直接关联articles

### P1 - 右侧栏精简
1. 右侧栏去重逻辑
2. 关键词云形式展示
3. 点击只高亮拓扑节点

### P2 - AI分析增强
1. AI分析按钮
2. AI分析面板
3. 增量存储分析结果

## 需要确认的问题

1. **日报时间线的展示数量**: 默认展示多少条？支持分页/无限滚动吗？
2. **AI分析的触发时机**: 用户点击按钮后立即调用AI？还是后台预生成？
3. **拓扑图连线的动画效果**: 具体想要什么样的动画（生长动画/淡入淡出/其他）？
4. **关键词云的展示**: 展示多少个关键词？如何确定字号大小？
5. **日报时间线和右侧栏的去重**: 基于article ID去重，还有其他维度吗？
